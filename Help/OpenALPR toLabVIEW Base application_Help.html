OpenALPR-base.vi

This project is help to integrate the OpnelALPR open source ALPR system to LabVIEW (for LV 2014 or newer versions). Everything is documented in the LV and C++ codes, if you want to find more information about the project, what this readme contains. The project also contains a html help for the example LW program, which could be find in the Help directory in the project.

<b>Important!!!</b>

The „openalpr_32” directory contains the 2.3 version of the OpnelALPR open source ALPR system. For the sucesfull operation please copy the „openalpr_32” directory to C:/ in a folder (or whether you want). And create a "OPENALPR_PATH" environment variable, which have to point to the directory which contains the "openalpr_32" directory. The environment variable’s value must be the path of the directory, which contains the „openalpr_32” directory, not the „openalpr_32” directory’s path itself. (eg. “C:\OpenALPR\alpr_32”, where the “OPENALPR_PATH” environment variable’s value is “C:\OpenALPR”)

 Default Country

Set the default country or region for the recognisation(for example: "hu" for Hungary). Default is "hu".

Max Possibility

An int type of data, witch contains how much possibilities we want to receive for every recognised licence plate with the possibility percentage.

image path to recognise

The path of the image, which will be recognised.

The pragram rearrange it as a string, and replace every / with \, so the dll can do the work with the passed string as a path.

Init Return

Returns the error value of the initialization. "OK " if no errors occurs. "ERR_LOAD" when the loading occurs errors.

Place of the Image

The string type path (with / instead of \) of the image, which have been recognised.

Detected Licence Plates as String

Returns all detected plates in a single string. Could be "ERR_LOAD" or "ERR_NO_PLATE" if any errors occurs.

The returned string's building pattern:

- the first number is information about the number of recognised plates in the picture.

- after the % symbol this is the same pattern all for the recignised plate:

-- Plate_n: the n is number of the recongnised plate.

-- recognisedplate_x%, : the plates are arranged in possibility prom the hightest to the lowest. In this pattern the recognisedplate is the plate's possible string(e.g.: ABC-123) without the "-" token(so the output in our example is ABC123). After the "_" tofen we can find the possibility percentage. afer every plate possibility there are a "," and after every recognised plate are a ";" token.

Processed Recognise Data:

Review the recognised plates and the plate ID's possibilities as a LabVIEW 2D Array. You can save this data to files or use it in your application as a normal Array.

    Every column and row contains 1 type of data. In every first row, the first column contains the date and time, the another columns are containing 1 and 1 plate possibility, with the possibility percentage in every second row.
